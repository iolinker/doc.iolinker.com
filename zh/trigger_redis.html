# Redis 触发器

通过监听 Redis 列表的新增数据来触发工作流。当数据通过 LPUSH 或 RPUSH 被推送到指定的 Redis 列表键时，工作流会自动执行。适用于构建消息驱动的工作流、任务队列和实时事件处理。

# 入参

  • 凭据（必填）：选择一个 Redis 凭据。可在【凭据管理】中创建 Redis 凭据类型，需配置以下选项：

    • Host：Redis 服务器地址。
    • Port：Redis 服务器端口（默认：6379）。
    • Auth：Redis 认证密码（可选）。

  • Method（必填）：要监听的 Redis 列表操作方式。支持：

    方法 说明
    lpush 监听从列表左侧推入的数据（LPUSH 操作）
    rpush 监听从列表右侧推入的数据（RPUSH 操作）

  • Select（必填）：Redis 数据库索引（从 0 开始，必须 ≥ 0）。

  • Key（必填）：要监听的 Redis 列表键名。

# 出参

每次有新数据被推送到指定的 Redis 列表时，工作流将被触发，推送的数据作为输出。输出为推送数据解析后的 JSON 内容。

{
  "data": "pushed-value"
}

实际输出格式取决于推送到 Redis 列表中的数据内容。

# 工作原理

  1. 当工作流启用时，IOLinker 连接到 Redis 服务器并监听指定的列表键。
  2. 当有新数据通过 LPUSH 或 RPUSH（取决于配置的方法）推送到列表时，该数据会被消费并触发工作流。
  3. 每条推送的数据触发一次独立的工作流执行，数据内容作为触发器输出。
  4. 数据从列表中被消费（弹出），因此不会重复触发工作流。
